Hello, and welcome to programming in Java!
This is the first of many tutorials for programming in Java and is mostly a "let's get things going" tutorial.
First of all, you are going to need to download the Java Runtime Environment (JRE) and the Java Development Kit (JDK).
Secondly, you will need a program with which you can program in Java. I use Eclipse, but Oracle provides NetBeans.
Once you have installed those programs, you are ready to make some Java!
Now, let's get to a simple program to teach you the basic set-up of a Java program.
You're going to need to create a new Java project, and then you need to create a new Java class.
Call the class whatever you want. I am using ClassNameHere.
Because this is your first program, it is tradition to make it print "Hello, world!" to the console.
The code below does just that with the "System.out.println("Thing to print");" method.
Fill the class with the following code:
//This is called a comment. Keep notes with them!
/*
*This is a comment, too. It allows for multiple lines.
*/
public class ClassNameHere { //This is called a class. It's the main part of a program.
  public static void main(String[] args) { //This is the main method. It is needed for all programs in at least one class. A "main" class.
    //Insert the "meat" of the program here. Because it's your first program, we'll do something simple.
    System.out.println("Hello, world!"); //This is your first command. It calls a method called "println" from the "System" class.
    //This tells the computer to print whatever is inside the parentheses into the console. In our case, it will print "Hello, world!".
    //Remember, it will not physically print the quotation marks. Those are used to tell the computer you are defining a String (of characters), like a sentence.
    //We'll get into types of variables later. For now, you have your first Java program!
  }
}

**This tutorial assumes you have followed the first Java Tutorial**

As you may know, I am creating a game called "Grundink." It has mod support, but that's worthless without knowing how to make mods, isn't it?
To make mods, you need to be able to code in Java (see the Java Tutorial 1 to set up a way to code in Java) and understand some basic Java syntax.
Now that you are able to code in Java, you will need to download Grundink. Follow this link and download it.
So you've got Grundink. Now what?
Now, you need to extract the game file itself. Unzip the Grundink Launcher you just downloaded any way you want (7zip, WinRAR, etc.) and navigate to classes.zip.
Open classes.zip and then extract Grundink.jar to a dedicated area for programming mods. Remember where you put it. It will come back soon.
Now, open up you Java IDE (Eclipse, NetBeans, etc.) and create a new Java project.
Create you main package and a new class. Now, (for Eclipse) right click on your project, select "Properties", and "Java Build Path".
Choose the "Libraries" tab, and choose "Add External JARS". Navigate to wherever you put the extracted Grundink.jar file and select it.
Congratulations, the hard part is over! You can now make mods for Grundink!

Grundink operates on a "need to know basis" so to speak. You mod will only be called upon when Grundink or your mod requests it.
These requests are made through Events and these are the backbone to Grundink.
For your first mod, we'll do something simple. It will be a mod that prints to Grundink's log, "Hello, world!" as soon as it is loaded.
package [mod.package.here];
import grundink.src.Grundink;
import grundink.src.Mod;
public class [ModName] extends Mod {
 @Override
 public void onInitialize() {
  Grundink.log("Hello, world!");
 }
}

 Now that you've coded the mod, you need to set it up for use with Grundink.
Run the Grundink Launcher you first downloaded and play the game. Exit the game whenever you'd like.
Doing that built all of the files Grundink needs and allows you to continue making your mod.
Navigate to your home directory (go to C:\Users\[User] on Windows, /home/[user]/ on Linux, etc.)
Open the folder called ".grundink" (you may have to show hidden files) and the folder "mods" inside of that.
Now, create a folder for your mod. Call it whatever. Create a new folder called "classes".
Then, create a file inside the folder above "classes" called "main_class.info" (/.grundink/mods/[theFolderYouMade]/main_class.info)
Lay the file out like so:
name:[mod_name]
main-class:
author:[author]
version:[mod_version]

 We'll come back to that file later; don't do anything with it.
Now, go back to your Java IDE and build the mod. Export it as a ".class" file to the "classes" folder.
If you gave your mod a package, put the class file inside folders that correspond to the package, ie. for a package "mod.src", the class would be in "/.grundink/mods/[theFolderYouMade]/classes/mod/src/Main.class"
Once the class is where it should be, go back to the "main_class.info" file and where it says "main-class:", put the path from the "main_class.info" file to the mod's main class.
Continuing with the above "mod.src" package example, the "main-class:" should have "mod.src.Main" (excluding quotation marks) after it. Now, if you did everything correctly, when you run Grundink, the mod will initialize and "Hello, world!" will be logged to "/.grundink/log.txt".
Here is the final main_class.info example file:
name: My_First_Mod
main-class: mod.src.Main
author: redusk
version: 1.0

 Remember to replace the values with the values that are correct for YOUR mod, especially the "main-class" path.
Also, remember that the "main-class" path uses periods (.) to separate folders and excludes the file type (.class) from the end.
